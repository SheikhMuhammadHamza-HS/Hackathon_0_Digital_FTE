## 1. Purpose
To build a "Minimum Viable Agent" (Digital FTE) that operates on a local-first architecture. It establishes the "Perception → Reasoning → Memory" loop without external autonomous actions.

## 2. Feature Set (MVP)
1.  **Folder Structure:**
    - Create `/Inbox`, `/Needs_Action`, `/Done`, and `/Logs`.
2.  **The Brain (Obsidian):**
    - `Dashboard.md`: A live-updating view of current tasks.
    - `Company_Handbook.md`: A ruleset file defining behaviors.
3.  **The Sense (Watcher):**
    - Script: `filesystem_watcher.py`.
    - Function: Monitors `/Inbox`, detects new files, generates metadata trigger in `/Needs_Action`.
4.  **The Reasoning (Claude Code):**
    - Reads `/Needs_Action`, updates `Dashboard.md`, moves original file to `/Done`.

## 3. User Roles
- **Human:** Drops files into `/Inbox`, reviews Dashboard.
- **Agent:** Reads `/Needs_Action`, Writes to `Dashboard.md` and `/Done`. Cannot delete files permanently.

## 4. Technical Requirements
- **Language:** Python 3.10+
- **Libraries:** `watchdog` (for file monitoring).
- **Environment:** Must load API keys from `.env`.
- **Latency:** File detection within 5 seconds.

## 5. Data Contracts

### A. Trigger File Schema (in /Needs_Action)
Filename format: `TRIGGER_{timestamp}.md`
Content:
---
type: "file_drop"
source_path: "path/to/file.pdf"
timestamp: "ISO-8601-Time"
status: "pending"
---
## Context
File detected in Inbox.

### B. Dashboard Format (Dashboard.md)
# Agent Dashboard
| Time | Task | Status |
|------|------|--------|
| 10:00| File1.pdf | Done |

## 6. Acceptance Criteria
1.  **Setup:** Running the setup script creates all folders.
2.  **Detection:** Dropping a file in `/Inbox` creates a corresponding `.md` file in `/Needs_Action`.
3.  **Completion:** Claude Code successfully updates the Dashboard table after processing.
